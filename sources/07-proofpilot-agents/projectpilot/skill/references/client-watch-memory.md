# ProjectPilot Client Watch Memory

This is not a blame file.
It is PM watch memory: the quiet manager-level reminders about what has gone wrong before, what tends to slip, and what deserves a little extra attention until the lane becomes reliable.

Rule:
- do not fixate on old mistakes
- do use prior misses to watch the next cycle more intelligently
- when a pattern stabilizes and stops recurring, downgrade it from active watch memory

---

## Pelican Coast Electric (PCE)
### Active watch memory
- Live pages can publish with wrong phone links, template residue, and missing service/schema details.
- Dead or misleading task links can make work look missing even when the live page exists elsewhere.
- Review queues can look stale for a long time, so ProjectPilot should avoid noisy escalation unless there is fresh movement or an actual deadline risk.
- Commercial location batches can get moved to approved before the live commercial URLs and publish-layer cleanup are actually clear.
- Shared publish-layer misses can travel as a batch here, especially wrong reused featured / OG images and footer dial-link mismatches.
- Cross-client review assets can bleed into Pelican tasks, and in the worst case a task can even get approved off the wrong client preview URL.
- Approved status can drift ahead of reviewer truth here, so the latest ClickUp comment may still reopen the work even when the task label says approved.

### What to watch for next
- when something is published, verify the live URL, phone/CTA behavior, and obvious template leftovers
- if the task link looks dead, recover the real live URL before calling it blocked
- focus on true delta, not stale queue totals
- treat launch verification as essential because this lane has shown real publish-layer misses before
- if a commercial page batch moves fast, make sure status approval is tied to the corrected live commercial URLs, not old general pages or draft links
- before treating a `ready for review` blog or page as a reviewer-delay issue, verify that the linked review asset actually belongs to Pelican Coast and not another client preview. Cross-client preview-link mixups can burn reviewer time and look like backlog when the real issue is owner/coordinator handoff cleanup
- before treating `approved` as real approval, compare the newest reviewer comment against the status. If the latest comment says one fix remains, keep it in correction until comment truth and task status match

### PM interpretation
This lane benefits from calm, precise live verification more than repeated backlog alarms.

## How to maintain this file
Add a watch-memory note when:
- a miss or near-miss repeats
- a specific client lane shows the same weak point again
- a person/client combination clearly benefits from a quiet extra reminder next cycle

Remove or downgrade a watch-memory note when:
- the pattern has clearly stopped recurring
- the process was fixed and stayed fixed across multiple cycles
