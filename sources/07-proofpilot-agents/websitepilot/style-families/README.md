# WebsitePilot Style Families

Style families are the missing layer between brand extraction and template execution.

They are not agencies.
They are not one-off clones.
They are reusable visual systems that turn strong references into repeatable ProofPilot output.

Each family has three parts:

1. `FAMILY.md` — when to use it, when not to use it, and the non-negotiables
2. `starter/index.css` — token, spacing, and motion defaults for the family
3. `starter/sections.tsx` — reusable section patterns that Website Brain can adapt into a live demo

## The current families

- `heroic-branded-conversion`
- `operator-proof-longform`
- `premium-outdoor-editorial`
- `clean-recurring-service`

## How this fits into the pipeline

1. Brand Brain extracts the truth about the client.
2. Style-family selection classifies the right visual lane.
3. Designer Brain commits the palette, typography, motif, transitions, and section rules inside that lane.
4. Website Brain picks the best template variant inside that lane and applies the design spec.

## Important rule

These families are inspired by strong references in the market, but the output must still be:

- client-branded
- asset-safe
- custom in feel
- different enough to pass the remove-the-logo test

Never lift another brand's logo, custom illustrations, branded characters, or proprietary copy.
Use the family's structural and stylistic logic, then rebuild it around the client's real brand.
