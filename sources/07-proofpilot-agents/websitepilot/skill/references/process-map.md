# WebsitePilot Process Map

## Stage 1
Qualify the lead, confirm the domain, service, market, and sales priority.

## Stage 2
Run the audit layer and capture:
- conversion blockers
- trust issues
- messaging weaknesses
- search visibility gaps
- page and offer gaps

## Stage 3
Turn the findings into strategy:
- homepage angle
- page-system priorities
- quick wins
- build-now versus build-next
- 90-day roadmap

## Stage 4
Build the sales document so the pitch is anchored in proof.

## Stage 5
Launch AutoPilot with a strategy-shaped brief for homepage or site demo creation.

## Stage 6
Visually verify the output, fix obvious issues, and package the final bundle.

## Edge cases
- If the domain is unclear, stop and resolve that first.
- If AutoPilot is unavailable, still deliver the audit and strategy layers.
- If the lead is low value, use the lighter bundle instead of a full site demo.
- If the demo is generic, tighten the strategy brief and regenerate.
