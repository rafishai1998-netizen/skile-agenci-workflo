# Report Types and Default Flow

## Full Monthly Report
Use when a formal monthly client report is needed.

### Default inputs
- prior month performance window
- previous full month as compare window unless otherwise requested
- current work completed list
- ranking / visibility context if relevant
- screenshots for sections where live API access is limited

### Default output sections
1. Cover and reporting window
2. Executive summary
3. KPI snapshot
4. Organic performance movement
5. Top pages and query themes
6. Work completed
7. Strategic insights
8. Next-period priorities
9. Appendix or screenshots

## Ad Hoc Performance Report
Use for a pulse update, leadership check-in, or urgent client question.

### Best format
1. What changed
2. Why it matters
3. What likely contributed
4. What should happen next

## Single-Layer Report
Use when the user only wants one slice of the picture.
Examples:
- SEO only
- GBP only
- content delivery only
- ranking movement only

### Best format
1. Scope statement
2. Core metrics or observations
3. Insight summary
4. Recommended action

## Recovery Report
Use when the normal report was delayed, partial, or needs rebuilding.

### Best format
1. Current truth
2. Covered period
3. Missing pieces or caveats
4. What changed since the missed report
5. Recommended next steps

## Default Build Flow
1. Lock client, period, and report type
2. Pull prior context and recent movement
3. Pull live data
4. Normalize and reconcile numbers
5. Decide the true story
6. Draft the report
7. Run QA checklist
8. Verify final link or file
9. Deliver concise Slack summary with link
