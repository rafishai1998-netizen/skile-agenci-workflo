# ReportPilot QA Checklist

Use this before delivering any report.

## 1. Data Integrity
- Reporting window is correct
- Compare window is correct
- GSC, GA4, and ranking data are not being mixed as the same metric
- Totals, deltas, and percentages reconcile
- Charts match the written narrative
- No placeholder or guessed numbers remain

## 2. Context Integrity
- Prior approvals, edits, and in-thread updates were considered
- Moving items are not described as stuck
- Delivered work is separated from in-review work
- Package changes or budget expansions are reflected where relevant
- Old issues already handled are suppressed

## 3. Strategic Integrity
- The report surfaces the 2 to 4 most important insights
- It distinguishes broad momentum from one-page or one-query spikes
- It distinguishes leading indicators from lagging outcomes
- It includes clear risks or watch-items, not just wins
- It ends with concrete next-step priorities

## 4. Client Integrity
- Correct client name, month, and domain everywhere
- Correct screenshots and labels
- No cross-client residue from prior report templates
- Tone is clean and client-ready

## 5. Delivery Integrity
- Final link works
- Final file is the intended version
- Status language is true to reality
- If a report is live, the live URL was checked before saying done

## Severity Guide

### Critical
- Wrong month
- Wrong client
- Wrong metrics or invented numbers
- Broken final link
- Report says delivered when it is not

### Warning
- Weak insight framing
- Unclear caveat on missing data
- Chart and text use different wording for the same movement
- Too much noise and not enough signal

### Improvement
- Sharper executive summary
- Better grouping of insights
- Clearer next-step recommendations
