# PM Message Patterns

## Pattern 1: Review batch plus one older item
Use when a lot of content just moved into review but only one item actually needs a nudge.

Example:

Jo Paula just moved *14 new commercial content items* into ready for review this afternoon, so Marcos's queue got a lot heavier all at once.

That batch looks active. The one older item still worth pulling back up is the *March SEO Report*. Jo Paula shared the lovable link on April 3. If that thread still has no final approval, that is the cleaner before-the-weekend review ask.

## Pattern 2: Same queue, but some of it already moved
Use when the big batch is no longer the issue because review already started.

Example:

Good news, part of the commercial batch is already moving. Marcos reviewed several of the new commercial items and sent edits back to Jo Paula.

The report itself is the one still sitting. If that thread still has no final sign-off, that is the item worth a final-look nudge before the weekend.

## Pattern 3: Deliverable with hidden edit loop
Use when a top-level message says delivered, but thread replies show more work.

Example:

Surface-level it looks submitted, but the thread still shows feedback on the delivered version. Status is *needs edits* or *waiting on final review*, not done.

## Pattern 4: Workload balancing note
Use when the insight is planning value, not blockage.

Example:

Worth planning around, Jo Paula dropped a large commercial batch into review today. Even if nothing is blocked, the queue changed enough that Marcos's afternoon review load is materially different now.

## Pattern 5: Soft pressure without overclaiming
Use when you want to resurface something without assuming nobody already handled it elsewhere.

Example:

If it's already been approved and sent, ignore this. If not, this looks like one of the cleaner items to knock out before the weekend because it's been sitting about a week.

## Quick Checklist Before Sending
- Did I read the latest thread replies?
- Is the item still open?
- Did someone already start handling it today?
- Am I naming the real owner, reviewer, and coordinator?
- Does this message help the person plan, decide, or act right now?

## Pattern 6: Warm stuck-task check-in
Use when a task has been in progress long enough to deserve a nudge, but the tone should reduce friction, not create it.

Example:

Hey <@USER>, this one's been in progress for a bit. Are you blocked on anything, or is it close to done? Just want to make sure you have what you need.

## Pattern 7: Ready-for-review reminder
Use when a reviewer-specific nudge is appropriate after about 48 hours with no movement.

Example:

<@REVIEWER>, looks like *{task name}* has been in ready for review for a couple days. Whenever you get a chance. If you want, I can pull the thread context for you.

## Pattern 8: Monthly task still untouched after the 15th
Use for ProofPilot monthly work that has become a timing risk.

Example:

Hey <@OWNER>, just noticing *{task}* for *{client}* hasn't been picked up yet. It's past the 15th so I wanted to flag it. <@U0AKNBHEG8L> <@U09CQS8HMEG> FYI. No pressure, just want to make sure it's on radar.

## Pattern 9: SEO report overdue support-first nudge
Use when a report is late enough to matter, but you want to offer help instead of just pressure.

Example:

<@OWNER>, the *{client} SEO report* for *{month}* is past the 7-day mark. If you're buried, I can help pull the GA4 and GSC data to save you time. Just let me know what would help most.

## Pattern 10: Something should be a task but isn't
Use when Slack intent exists but no ClickUp task exists yet.

Example:

Noticed in Slack on *{date}* that *{person}* mentioned *{thing}*. I don't see a ClickUp task for it. Want me to create one, or is it already being handled somewhere else?

## Pattern 11: Same-day multi-client rollup
Use when several issues exist, but only one deserves the headline.

Example:

A few clients need attention today. I posted the details where they live. The one I'd prioritize first is *{most urgent item}* because *{one-sentence why}*.

## Pattern 12: SEO report early-month writer check-in
Use on day 4-5 when the report is not yet in progress and you want one gentle nudge, not escalation.

Example:

Hey <@OWNER>, just wanted to make sure the *{client}* report is on your radar for this month. If you need anything pulled on the data side, I can help.

## Pattern 13: SEO report manager escalation
Use on day 6-7 business days when the report is still not delivered and the risk is now client-facing.

Example:

The *{client}* report still has not been delivered and we're at day 7. Flagging it here so it doesn't start affecting the client relationship. If helpful, I can pull the GA4 organic summary and GSC query data right now.

## Pattern 14: GBP ready-for-review but actually waiting in Slack
Use when ClickUp says ready for review, but the real blocker is approval-channel inertia.

Example:

This looks less like a writing delay and more like an approval-lane delay. The GBP task is sitting in *ready for review*, and the actual bottleneck appears to be in *#for-approvals*, not ClickUp.

## PM communication anti-patterns to avoid
- Em dashes
- Harsh overdue language like "This is overdue. Complete it."
- Calling something complete while QA or approval is still pending
- Re-flagging the same thing two days in a row with no new angle
- Using channel-wide alarm language when a reviewer- or owner-specific nudge would do the job
