# DESIGN REQUIREMENTS
Every website we deliver needs to hit these standards. Three things together make a site feel unique: Cohesive, Detail, and Dynamic. Study the examples above to see how these elements come together.
## 1. Cohesive
The site should have a unified look and feel. Every element should work together as part of a single visual system.
### Color Application
Colors need to be applied with intention. Primary colors for key elements, secondary colors for accents, and a clear hierarchy throughout. Think about color blocking, overlays on images, gradient usage, and how color draws attention to CTAs and important content.
### Typography
Font choices should reflect the brand personality. Consider the pairing between headline and body fonts, letter spacing, font weights for hierarchy, and how type creates visual interest. Headlines should have presence. Body text should be readable. The combination should feel intentional.
### Photo Treatment
Photos should feel like part of the design, not just dropped in. Consider color overlays, consistent cropping styles, custom frames or masks, and how images integrate with surrounding elements. The treatment should tie photos into the brand's visual identity.
### Brand Personality
The site should feel like it could only belong to this company. Every element should reinforce who they are. If you removed the logo, the design itself should still communicate the brand's personality, industry, and values.
## 2. Detail
Custom elements that make the site feel designed, not assembled. These are the touches that separate a template from a real design.
### Custom Icons
No generic icon packs. Icons should match the brand's style, color palette, and overall visual language. They should feel like they were made for this company. Consider line weight, fill style, and whether they tie into any brand motifs or themes.
### Patterns, Textures, and Background Elements
Custom visual elements in the background add depth and make the site feel designed. This includes subtle textures, geometric patterns, line work, gradients, shapes, or industry-specific motifs. These elements should complement the content without competing with it.
### Borders, Dividers, and Section Treatments
How sections are separated matters. Consider custom dividers, angled section breaks, overlapping elements, or unique border treatments. Rounded corners vs. sharp edges should be consistent and intentional. These details signal that someone designed this, not just assembled blocks.
### Buttons and Interactive Elements
Buttons should look designed, not default. Consider shape, padding, border radius, shadow, hover states, and how they fit the overall aesthetic. Form fields, toggles, and other interactive elements should follow the same visual language.
## 3. Dynamic
The site should have a sense of movement and rhythm. It should feel alive, not static.
### Section Layout and Flow
Each section should be thoughtfully composed. Consider the rhythm of the page, how content is grouped, the balance between text and visuals, and how the eye moves through the layout. Avoid stacking generic blocks. Create intentional compositions that guide the visitor.
### Visual Rhythm
Alternate between different section types. Vary the pacing. Use whitespace strategically. The page should have peaks and valleys that keep visitors engaged as they scroll. Predictable, repetitive layouts feel static.
### Movement and Animation
Subtle animations on scroll, hover effects, and micro-interactions add life to the site. These should be purposeful and restrained. The goal is to create energy without distraction. Elements should feel like they respond to the visitor.
