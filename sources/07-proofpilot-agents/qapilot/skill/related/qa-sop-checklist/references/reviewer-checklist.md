# Reviewer Checklist

Use this worksheet while auditing a page.

## Preflight
- [ ] Live URL captured
- [ ] Client captured
- [ ] Target keyword or topic captured
- [ ] Page type identified
- [ ] Intended audience identified
- [ ] Brief / task context reviewed
- [ ] 1 to 3 sibling pages opened if template comparison is needed

## Phase 1: Orientation Before Reading
- [ ] I know who the page is for
- [ ] I know the page's one job
- [ ] I know what template/page family it belongs to
- [ ] I know the recurring bugs I expect on this site
- [ ] I know what a corrected sibling page looks like, if relevant

## Phase 2: Structural Scan
- [ ] No duplicate sections/headings/cards
- [ ] No wrong-content sections for this page type or audience
- [ ] No obvious template residue from another page
- [ ] No obvious missing required sections
- [ ] Structure matches the intended job of the page

## Phase 3: Sentence-Level Grammar
- [ ] No comma splices
- [ ] No fragments
- [ ] `-ing` phrases attach cleanly, usually with a comma when needed
- [ ] List introductions have a colon or comma before the list begins
- [ ] No run-ons
- [ ] Subject-verb agreement is clean
- [ ] No suspicious double spaces
- [ ] Form intro copy is not hedging or outdated

## Phase 4: Audience and Tone
- [ ] Copy matches the intended audience
- [ ] Commercial pages do not use residential cues like `your family`, `your home`, `homeowners`
- [ ] Local details are accurate
- [ ] Tone is consistent across sections
- [ ] The page still does its one job well

## Phase 5: Link and Technical
- [ ] Anchor text matches href destination
- [ ] No wrong-domain or competitor links
- [ ] Displayed phone numbers match `tel:` hrefs
- [ ] Phone links are clickable and correctly formatted
- [ ] Title tag checked
- [ ] Meta description checked
- [ ] Canonical checked
- [ ] Schema checked
- [ ] Featured image / `og:image` checked
- [ ] Internal links checked
- [ ] WhatConverts numbers only flagged if broken/misrouted/malformed/non-clickable

## Phase 6: Cross-Page Pattern Recognition
- [ ] I checked the site's known recurring failure zones first
- [ ] I carried any sibling-page fixes across this page family
- [ ] I prioritized high-impact issues before cosmetic ones

## Final Verdict
- [ ] Audience and page purpose confirmed in the writeup
- [ ] Highest-impact structural/context issue called out first
- [ ] Highest-impact technical/link issue called out next
- [ ] Most important grammar/style fix called out after that
- [ ] Verdict assigned: pass / needs edits / fail before approval
