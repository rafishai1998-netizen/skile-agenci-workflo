# Agent Orchestration

> Multi-agent coordination with swarm orchestration, SPARC methodology, DevFleet team building, hooks automation, and collaborative agent patterns.

## When to Use These Skills
- Coordinating multiple AI agents to work on a complex task
- Implementing the SPARC methodology for structured development
- Setting up DevFleet teams or swarm topologies
- Automating workflows with hooks and event-driven agent triggers

## Start Here
If you're new to this category, start with **swarm-orchestration** for multi-agent coordination or **sparc-methodology** for a structured development approach. **team-building** is useful for understanding how to compose agent teams before diving into advanced orchestration.

## Skill Relationships
**swarm-orchestration** is the core skill that manages agent lifecycles and communication in a swarm topology. **sparc-methodology** provides a structured framework (Specification, Pseudocode, Architecture, Refinement, Completion) that can drive multi-agent workflows. **devfleet** and **team-building** handle the composition layer -- deciding which agents to spawn and how to organize them. **agent-topology** defines how agents connect (hierarchical, mesh, ring), while **consensus-patterns** handles agreement when multiple agents produce conflicting outputs. **hooks-automation** enables event-driven triggers that launch agents automatically. **task-decomposition** feeds into all orchestration skills by breaking down work into agent-sized units.
