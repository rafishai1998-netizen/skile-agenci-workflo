# GitHub CI/CD

> GitHub workflow automation, code review swarms, release management, project management, and multi-repo coordination.

## When to Use These Skills
- Automating GitHub Actions workflows and CI/CD pipelines
- Running AI-powered code review on pull requests
- Managing releases, changelogs, and versioning
- Coordinating work across multiple repositories

## Quick Reference

| Skill | Difficulty | Trigger Phrase | Best For |
|-------|-----------|----------------|----------|
| github-code-review | Intermediate | `/github-code-review` | AI-powered PR code review |
| github-workflow-automation | Intermediate | `/github-workflow-automation` | GitHub Actions workflow setup |
| release-management | Intermediate | `/release-management` | Versioning, changelogs, and releases |
| project-management | Beginner | `/project-management` | GitHub Projects and issue tracking |
| multi-repo-coordination | Advanced | `/multi-repo-coordination` | Cross-repository workflows and sync |

## Start Here
If you're new to this category, start with **github-code-review** for PR review automation or **github-workflow-automation** for CI/CD pipeline setup. These are the two most commonly needed GitHub automation skills.

## Skill Relationships
**github-workflow-automation** provides the CI/CD foundation that other skills build on -- it sets up the pipelines where code review, testing, and releases run. **github-code-review** integrates into PR workflows created by the automation skill. **release-management** depends on CI/CD pipelines for automated releases and changelog generation. **project-management** tracks the work that flows through the CI/CD pipeline. **multi-repo-coordination** extends all other skills across repository boundaries, handling cross-repo dependencies, synchronized releases, and unified CI/CD strategies.
