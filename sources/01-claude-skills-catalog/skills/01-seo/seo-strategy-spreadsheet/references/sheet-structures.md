## 3. Buyer Personas

**Purpose**: Define 3-5 customer personas to guide content strategy.

### Persona Template

For each persona, create a section with:

**[PERSONA NAME] - [Tagline]**

| Attribute | Details |
|-----------|---------|
| Demographics | Age, income, location, homeowner status |
| Primary Pain Point | Main problem they're trying to solve |
| Secondary Pain Points | Additional concerns |
| Buying Triggers | What prompts them to seek service |
| Decision Factors | Price, speed, trust, reviews, etc. |
| Common Questions | FAQs this persona asks |
| Content Preferences | Blog, video, calculator, etc. |
| Search Behavior | Keywords they use |
| Objections | What might stop them from buying |

### Example Personas for Electrician

1. **"Budget-Conscious Betty"** - First-time homeowner watching costs
2. **"Safety-Worried Sam"** - Concerned about older home's wiring
3. **"Upgrade-Ready Ryan"** - Planning EV charger or panel upgrade
4. **"Emergency Ethan"** - Needs immediate help
5. **"Renovation Rita"** - Major home project underway

**Column widths**: A=180px, B=400px

### Formatting Rules

1. **Main Service rows**: Navy background, white bold text, ALL CAPS
2. **Sub-Service rows**: White background, black text, → prefix in Page Name
3. **Section headers**: Navy background, white bold, merge across all columns
4. **Alternating rows**: White and gray (#F4F4F4) for data rows
5. **Status cells**: Color-coded per status value
6. **Funnel cells**: Color-coded (Blue=TOF, Amber=MOF, Red=BOF)
7. **Priority cells**: Color-coded (Green=HIGH, Lime=MEDIUM, Gray=LOW)

### Pillar-Based Organization Pattern

Blog content is organized by PILLARS. Each pillar represents a major topic cluster with supporting blog posts underneath.

**PILLAR ROW FORMAT** (Navy background, white bold text, merged across columns):
```
PILLAR: [TOPIC NAME IN ALL CAPS]
```

**Supporting posts** appear below each pillar with white or alternating gray backgrounds.
