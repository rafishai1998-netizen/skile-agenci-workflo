# Language-Specific

> Patterns, testing, code review, and build tools organized by programming language -- covering TypeScript/JS, Python, Go, Rust, Kotlin, Java/Spring Boot, C++, Swift, Perl, Flutter, Laravel, and Django.

## When to Use These Skills
- Writing idiomatic code in a specific programming language
- Setting up testing frameworks and patterns for your language/stack
- Reviewing code against language-specific best practices
- Configuring build tools, linters, or package managers for a language

## Start Here
If you're new to this category, start with the **{language}-patterns** skill for your stack (e.g., **typescript-patterns**, **python-patterns**, **go-patterns**). It provides the foundational idioms and will reference testing and review skills when relevant.

## Skill Relationships
For each language, skills follow a consistent structure: **{lang}-patterns** teaches idiomatic code, **{lang}-testing** covers test setup and strategies, **{lang}-review** provides code review checklists, and **{lang}-build** handles toolchain configuration. The patterns skill is the foundation -- testing and review skills build on the conventions it establishes. Cross-language skills like those in the development-workflow category (TDD, pair programming, security review) complement these language-specific skills by providing methodology that applies across all languages.
