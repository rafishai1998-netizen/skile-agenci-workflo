# Memory & Databases

> AgentDB vector search, learning algorithms, memory patterns, optimization strategies, PostgreSQL, and ClickHouse expertise.

## When to Use These Skills
- Implementing vector memory and semantic search for AI agents
- Designing database schemas or optimizing query performance
- Setting up PostgreSQL or ClickHouse for production workloads
- Building memory persistence layers for agent systems

## Start Here
If you're new to this category, start with **agentdb-memory-patterns** for agent memory architecture or **postgres-patterns** for general database work. These two skills cover the most common use cases in their respective domains.

## Skill Relationships
**agentdb-memory-patterns** defines the foundational architecture for agent memory -- how memories are stored, indexed, and retrieved. **vector-search-patterns** provides the underlying search mechanism (HNSW indexing, similarity scoring) that AgentDB uses. **agentdb-learning** extends memory patterns with algorithms that improve retrieval quality over time. **agentdb-optimization** tunes the performance of the entire memory stack. **memory-persistence** handles the durability layer, ensuring memories survive agent restarts. **postgres-patterns** and **clickhouse-patterns** are standalone database skills that can serve as backing stores for agent memory or for any application database needs.
